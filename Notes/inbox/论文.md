# 参考文献列表
在 LaTeX 中定义参考文献列表格式，核心是**匹配论文模板的要求（字体、字号、编号样式、条目结构、间距等）**，主要分为「手动定义」（适合简单需求，如课程论文）和「宏包自动定义」（适合复杂格式，如期刊 / 学位论文）两种方式。以下结合你之前的北大课程论文模板，从「底层逻辑→具体操作→实例适配」三方面拆解，让你不仅会用，还懂原理。

## 一、先明确核心：你需要定义哪些格式要素？

无论哪种方式，定义参考文献格式本质是指定以下 5 个关键要素（对应课程论文模板要求）：

1. **标题格式**：如「参考文献」（宋体三号加粗，居中 / 左对齐）；
2. **编号样式**：如 [1]、[2]（方括号 + 数字，无点号）；
3. **条目格式**：
    - 字体字号：宋体五号；
    - 条目结构：不同文献类型（图书、期刊、网站）的字段顺序（如「作者：书名，出版社，年份，页码」）；
    - 间距：条目内单倍行距，条目间空半行；
4. **字段分隔**：如作者之间用「、」，出版信息之间用「，」（全角标点）；
5. **缩进**：条目首行无缩进，第二行起缩进 2 字符（或默认缩进）。

## 二、方式一：手动定义（适合课程论文，简单直接）

底层逻辑：用 LaTeX 原生的 `thebibliography` 环境，直接控制每个部分的格式，无需复杂宏包，适配你之前的模板。

### 1. 完整定义代码（直接对接北大课程论文模板）

latex

```latex
% 第一步：在导言区设置参考文献相关字体（提前统一格式）
\usepackage{ctex} % 中文支持
\usepackage{setspace} % 控制行距

% 第二步：正文末尾定义参考文献列表
\vspace{0.5cm} % 与正文的间距
% 参考文献标题（宋体三号加粗，左对齐，无编号）
\noindent \textbf{\Large 参考文献} % \Large≈三号，\textbf加粗，\noindent取消首行缩进
\vspace{0.3cm} % 标题与条目间距

% 开始参考文献列表（{99}表示最多支持99条，编号从1开始）
\begin{thebibliography}{99}
    % 关键设置：条目格式（宋体五号，单倍行距）
    \renewcommand{\bibitemsep}{0.5cm} % 条目之间空0.5cm（半行）
    \renewcommand{\itemsep}{0cm} % 条目内行距（单倍）
    \ctexset{font={\songti \small}} % 宋体五号（\small≈五号）

    % 条目1：图书（格式：作者：书名，出版社，年份，页码）
    \bibitem{1} 习近平：《习近平谈治国理政》第4卷，外文出版社，2022年版，第3-5页。

    % 条目2：期刊论文（格式：作者：文章标题，期刊名，年份，期号，页码）
    \bibitem{2} 吴敬琏：《改革开放40年的回顾与思考》，《中国社会科学》，2018年第10期，第4-18页。

    % 条目3：网站文献（格式：作者：文章标题，网站名，发布日期）
    \bibitem{3} 国务院新闻办公室：《新时代的中国改革开放》白皮书，中国政府网，2023年12月28日。

    % 条目4：报纸文章（格式：作者：文章标题，报纸名，出版日期）
    \bibitem{4} 人民日报评论员：《坚定不移推进全面深化改革》，《人民日报》，2024年1月1日，第1版。
\end{thebibliography}
```

### 2. 关键命令拆解（懂原理才会灵活调整）

|命令|作用|底层逻辑|
|---|---|---|
|`\begin{thebibliography}{99}`|启动参考文献列表|`{99}` 是编号的「占位宽度」（比如 99 条以内编号都是两位数，排版对齐），若超过 100 条，改为`{999}`|
|`\renewcommand{\bibitemsep}{0.5cm}`|调整条目间距|原生`thebibliography`条目间距较窄，用此命令重定义（0.5cm≈半行，匹配模板要求）|
|`\ctexset{font={\songti \small}}`|统一条目字体|`\songti`= 宋体，`\small`= 五号，直接作用于所有`\bibitem`条目|
|`\bibitem{1}`|定义单个参考文献|`{1}`是引用标签（正文用`\cite{1}`引用），编号会自动按顺序生成 [1]、[2]|
|`\noindent \textbf{\Large 参考文献}`|标题格式|`\noindent`取消标题首行缩进，`\Large`+`\textbf`对应「三号加粗」|

### 3. 灵活调整技巧（适配不同模板）

- 若模板要求「编号带点号（1.、2.）」：在导言区添加 `\renewcommand{\bibnumfmt}[1]{#1.}`；
- 若要求「无编号（字母顺序）」：把`\bibitem{1}`改为`\bibitem{ref1}`（标签用字母 / 单词），同时去掉`{99}`（改为`{}`）；
- 若要求「条目首行缩进 2 字符」：在`thebibliography`环境内添加 `\setlength{\itemindent}{2em}`（1em≈1 字符，中文需 2em）。

## 三、方式二：宏包自动定义（适合复杂格式，如毕业论文 / 期刊）

底层逻辑：用专门的参考文献宏包（如 `biblatex`），通过「配置命令」批量定义格式，无需手动调整每个条目，更规范高效。

### 1. 常用宏包对比（选对工具）

|宏包|优点|适用场景|编译方式|
|---|---|---|---|
|`natbib`|兼容老模板，支持作者 - 年份 / 编号引用|课程论文、普通期刊|XeLaTeX + BibTeX|
|`biblatex`|灵活度极高，支持中文自定义，字段丰富|毕业论文、复杂格式期刊|XeLaTeX + Biber|

推荐用 `biblatex`（中文支持更好，适配性强），以下是完整示例：

### 2. `biblatex` 定义示例（对接课程论文模板）

#### 第一步：导言区配置宏包（定义格式规则）

latex

```latex
\documentclass{article}
\usepackage{ctex} % 中文支持
\usepackage{biblatex} % 核心宏包
\usepackage{setspace} % 行距控制

% 1. 导入参考文献数据库（单独的.bib文件，推荐用法）
% 新建一个「refs.bib」文件，存放所有参考文献条目（见下文）
\addbibresource{refs.bib}

% 2. 定义参考文献格式（核心配置）
\setcounter{biburlnumpenalty}{9000} % 避免网址换行错乱
\setcounter{biburlucpenalty}{9000}

% （1）标题格式：宋体三号加粗，左对齐
\defbibheading{bibliography}[\bibname]{%
  \noindent \textbf{\Large #1} \vspace{0.3cm}%
}

% （2）编号样式：方括号+数字（[1]、[2]）
\DeclareFieldFormat{labelnumberwidth}{\mkbibbrackets{#1}} % 编号加方括号
\setlength{\labelnumberwidth}{2em} % 编号宽度（避免排版拥挤）

% （3）条目格式：宋体五号，单倍行距
\setunit{\addcomma\space} % 字段分隔符（逗号+空格）
\ctexset{font={\songti \small}} % 条目字体
\renewcommand{\bibitemsep}{0.5cm} % 条目间距

% （4）自定义文献类型字段顺序（关键！）
% 图书：作者 → 书名 → 出版社 → 年份 → 页码
\DeclareBibliographyDriver{book}{%
  \usebibmacro{bibindex}%
  \usebibmacro{author/editor+others}% 作者
  \setunit{\addcolon\space}\newblock% 分隔符：冒号
  \usebibmacro{title}% 书名
  \setunit{\addcomma\space}\newblock% 分隔符：逗号
  \usebibmacro{publisher}% 出版社
  \setunit{\addcomma\space}\newblock% 分隔符：逗号
  \usebibmacro{year}% 年份
  \setunit{\addcomma\space}\newblock% 分隔符：逗号
  \usebibmacro{pages}% 页码
  \finentry% 结束条目
}

% 期刊论文：作者 → 文章标题 → 期刊名 → 年份 → 期号 → 页码
\DeclareBibliographyDriver{article}{%
  \usebibmacro{bibindex}%
  \usebibmacro{author/editor+others}%
  \setunit{\addcolon\space}\newblock%
  \usebibmacro{title}%
  \setunit{\addcomma\space}\newblock%
  \usebibmacro{journal}% 期刊名
  \setunit{\addcomma\space}\newblock%
  \usebibmacro{year}%
  \setunit{\addcomma\space}\newblock%
  \usebibmacro{issue}% 期号
  \setunit{\addcomma\space}\newblock%
  \usebibmacro{pages}%
  \finentry%
}
```

#### 第二步：创建 `refs.bib` 文件（存放参考文献数据）

单独新建一个文本文件，命名为 `refs.bib`，按以下格式填写条目（字段与宏包配置对应）：

bibtex

```bibtex
% 图书（类型：book）
@book{xi2022,
  author = {习近平},
  title = {习近平谈治国理政},
  volume = {4}, % 卷号
  publisher = {外文出版社},
  year = {2022},
  pages = {3-5}, % 页码
  note = {第4卷} % 补充说明
}

% 期刊论文（类型：article）
@article{wu2018,
  author = {吴敬琏},
  title = {改革开放40年的回顾与思考},
  journal = {中国社会科学},
  year = {2018},
  issue = {10}, % 期号
  pages = {4-18}
}

% 网站（类型：online）
@online{gov2023,
  author = {国务院新闻办公室},
  title = {新时代的中国改革开放},
  url = {http://www.gov.cn/zhengce/2023-12/28/content_5832677.htm},
  year = {2023},
  month = {12},
  day = {28},
  note = {中国政府网}
}
```

#### 第三步：正文中引用与生成列表

latex

```latex
\begin{document}
% 正文引用（按标签引用，自动生成编号[1]）
改革开放是当代中国最鲜明的特色\cite{xi2022}，其核心是解放和发展生产力\cite{wu2018}。

% 生成参考文献列表（自动应用导言区定义的格式）
\printbibliography[title={参考文献}] % title指定标题，覆盖默认的「参考文献」
\end{document}
```

#### 第四步：编译方式（关键！）

用 `biblatex` 必须按以下顺序编译（否则格式不生效）：

1. XeLaTeX（生成.aux 文件，记录引用标签）；
2. Biber（读取.refs.bib 文件，按配置生成格式）；
3. XeLaTeX（将格式化的参考文献插入文档）。

Overleaf 中可自动识别，本地编译（如 TeXstudio）需在「选项→配置 TeXstudio→构建」中设置默认编译器为 XeLaTeX，Biber 为参考文献工具。

## 四、两种方式的对比与选择建议

|对比维度|手动定义（thebibliography）|宏包定义（biblatex）|
|---|---|---|
|操作难度|低（直接写条目，无需配置）|中（需学习配置命令）|
|灵活度|低（修改格式需逐行调整）|高（批量配置，支持复杂格式）|
|适用场景|课程论文（≤20 条参考文献）|毕业论文、期刊论文（≥20 条）|
|引用方式|仅支持编号引用（\cite {1}）|支持编号 / 作者 - 年份引用（\cite {xi2022} / \textcite {xi2022}）|

### 对你的建议：

- 若写**课程论文**（如之前的北大 “习思想概论” 课论文）：用「手动定义」，简单高效，直接匹配模板要求；
- 若写**毕业论文**或**参考文献较多（≥20 条）**：用「biblatex 宏包定义」，更规范，后续修改格式（如换期刊模板）只需调整配置命令，无需改动条目。

## 五、常见问题排查（避坑指南）

1. **中文显示乱码**：确保编译方式为 XeLaTeX，且导入了 `ctex` 宏包；
2. **编号错乱**：手动定义时，`\bibitem` 标签无需按顺序（LaTeX 会自动排序），但建议按引用顺序命名；
3. **格式不生效**：宏包定义时，必须用 Biber 编译（而非 BibTeX），且 `refs.bib` 文件路径正确（与.tex 文件同目录）；
4. **标点符号错误**：中文参考文献需用全角标点（、，。），避免半角标点（, .）导致排版混乱。
