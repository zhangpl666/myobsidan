# Git概述
Git是一个免费开源的分布式版本控制系统

## 何为版本控制
版本控制是一种记录文件内容变化，以便将来查阅特定版本修订情况的系统

## 为什么要版本控制
***个人开发过渡到团队协作***

## 版本控制工具
### 集中式版本控制工具
集中化的版本控制系统，有单一的集中管理服务器，协同工作的人都通过客户端链接服务器

好处：每个人都可以看到项目中其他人正在做什么。而管理员掌握每个开发者的权限并管理一个集中化的版本控制系统。

缺点：如果服务器宕机一小时，那么在这一小时内，谁都无法提交更新，也就无法协同工作

### 分布式版本控制工具
每个客户端都是一个服务器

客户端提取的不是最新版本的文件快照，而是完整的把代码仓库镜像下来，这样任何一处协同工作用的文件发生故障，事后都可以回复

远程库：分布式版本控制工具有一个代码托管中心——远程库，通过远程库提取别人的代码和版本

## git工作机制

在工作区写代码，添加到暂存区，确定好后添加到本地库

工作区和暂存区的内容可以删除，不形成版本，一但上传到本地库形成版本，就不能修改删除了，除非删掉整个本地库   ~~简称删库跑路~~

![[git工作原理图.png]]

本地库可以推送到远程库中，别人就可以提取了

## 代码托管中心

代码托管中心是基于网络服务器的远程代码仓库，一般称为远程库

* 局域网（不开源）
	* GitLab
* 互联网（开源）
	* GitHub（外网，官方）
	* Gitee码云（国内）

## 分支
git版本控制的本质就是有一个HEAD指针和多个分支指针指来指去

版本控制中，同时推进多个任务，每个任务都可以创建一个分支

同时并行推进多个功能开发，提高开发效率
