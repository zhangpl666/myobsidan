# 关于需要知道的一些名词
注意这些后缀都是windows系统中的，Linux、MacOS不同
- 链接库有两种方式，一种是静态链接，一种是动态链接
- .lib文件扩展名：是静态库，里面包含了各种函数功能实现；
- .dll文件扩展名：是动态库，里面有功能实现
	- dll.lib文件扩展名：是动态库的索引，用于在.dll中找到需要的函数

动态和静态链接都需要有头文件！！（我一直混淆这个概念）
**头文件是「告诉编译器函数怎么用」，而 .lib（导入库）是「告诉链接器函数在哪个 DLL 里」**
# 静态链接
静态链接这种方式会将你链接的库的内容直接写入exe文件中，这意味着占用更大的内存，当然从技术上来说运行更快

实现静态链接，首先要在程序中写入头文件。不过程序当然是不知道头文件在哪的。右键project，属性中找到C/C++中的general，将你的.h路径写进附加包含目录（注意用分号分开！）

还要找到.lib在哪，打开project属性，链接器中general中，输入附加库目录；input中输入.lib文件名。这里，前者是告诉这个文件在哪，后者是告诉用哪个文件

# 动态链接
静态链接由于过于浪费内存，于是有动态链接。dll中的内容不会写入exe文件，而是在运行时到dll中找。动态链接的文件大小显著小于静态链接

跟静态链接配置基本相同，只不过这里.lib文件中没有函数实现

Windows 加载 DLL 时，会按以下顺序查找（默认规则，无编码修改），只要 DLL 在其中任意一个路径，就能被找到：

1. **EXE 所在目录**（你管那么多，给我放在这里就是了）
2. 系统目录：`C:\Windows\System32`（32 位 DLL 放这，64 位系统也优先搜这）
3. 系统目录：`C:\Windows\System`（兼容旧系统，少用）
4. Windows 目录：`C:\Windows`
5. 当前工作目录（运行 EXE 时的命令行所在目录，不是 EXE 目录，慎用）
6. 环境变量 `PATH` 中所有目录（推荐，适合多个程序共用 DLL）

# 自定义库
跟前面配置完全相同，只不过记得右键project找到生成依赖项，将要引用的库勾选上
编译时会将依赖项一起编译一遍，这样就可以避免依赖项修改产生的问题
